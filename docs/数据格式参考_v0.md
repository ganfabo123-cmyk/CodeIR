## 1. CodeIR 两个变体（item 5：先编一个，并留"信息无损度"实验轴）

设计哲学：IR 携带的信息越多，模型B(IR→码)越不用"重新推导算法"，级联损失越小；
但越接近伪代码，模型B 就越没活干、拆分越没意义。
**"精简 ↔ 无损"是一条光谱，本身就是一个可做的实验轴。** 故 v0 给两端各一个变体。

### 变体 L — Lean（精简版）

只固定"算法决策"，把"怎么写出来"留给模型B。信息密度低，考验模型B 的重建能力。

```yaml
pattern: dynamic_programming
key_insight: 以每个元素结尾的最长上升子序列长度，可由前面更小元素的状态转移得到
state: dp[i] = 以 nums[i] 结尾的最长递增子序列长度
transition: dp[i] = max(dp[j]+1 for j<i if nums[j] < nums[i])
complexity: { time: "O(n^2)", space: "O(n)" }
edge_cases: [空数组返回0]
```

### 变体 R — Rich / 尽可能无损（item 5 主推）

目标：模型B 几乎只需"翻译成 Python"，不需重新决策。比 L 多了 `signature`（消除 I/O 猜测）和 `algorithm`（有序步骤，近伪代码但语言无关）。

```yaml
signature: "def lengthOfLIS(self, nums: List[int]) -> int"   # 精确签名，消除 I/O 歧义
pattern: dynamic_programming
key_insight: 以每个元素结尾的最长上升子序列长度，可由前面更小元素的状态转移得到
data_structures:
  - "dp: List[int]，长度 n，dp[i] = 以 nums[i] 结尾的 LIS 长度"
algorithm:                              # 有序步骤 = 无损核心，模型B 照此翻译
  - "若 nums 为空，返回 0"
  - "初始化 dp = [1] * n"
  - "for i in 1..n-1: for j in 0..i-1: 若 nums[j] < nums[i]: dp[i] = max(dp[i], dp[j]+1)"
  - "返回 max(dp)"
complexity: { time: "O(n^2)", space: "O(n)" }
edge_cases:
  - "空数组 → 0"
  - "全降序 → 每个 dp[i]=1，结果 1"
```

> **L 与 R 的差**：`signature` + `data_structures` + `algorithm` 三块。
> 后续 IR 消融实验 = 逐块增减，看 pass@1 怎么变 → 量化"IR 给多少信息才够"。
> v0 训练**先用 R**（无损优先，保住效果），L 留作对照。

## 2. 完整三元组真实样例（题面 / CodeIR / 代码）

> 一条训练样本的"源头"。下面以 LIS 为例演示**字段长什么样**（真实数据用新题）。

```json
{
  "problem_id": "lc-300-demo",
  "difficulty": "medium",
  "statement": "给定整数数组 nums，返回其中最长严格递增子序列的长度。子序列是由数组删除若干（含0）元素、不改变剩余元素顺序得到的序列。约束：1 <= nums.length <= 2500，-10^4 <= nums[i] <= 10^4。",
  "signature": "def lengthOfLIS(self, nums: List[int]) -> int",
  "codeir": {
    "signature": "def lengthOfLIS(self, nums: List[int]) -> int",
    "pattern": "dynamic_programming",
    "key_insight": "以每个元素结尾的最长上升子序列长度，可由前面更小元素的状态转移得到",
    "data_structures": ["dp: List[int]，长度 n，dp[i] = 以 nums[i] 结尾的 LIS 长度"],
    "algorithm": [
      "若 nums 为空，返回 0",
      "初始化 dp = [1] * n",
      "for i in 1..n-1: for j in 0..i-1: 若 nums[j] < nums[i]: dp[i] = max(dp[i], dp[j]+1)",
      "返回 max(dp)"
    ],
    "complexity": {"time": "O(n^2)", "space": "O(n)"},
    "edge_cases": ["空数组 → 0", "全降序 → 1"]
  },
  "code": "class Solution:\n    def lengthOfLIS(self, nums: List[int]) -> int:\n        if not nums:\n            return 0\n        n = len(nums)\n        dp = [1] * n\n        for i in range(1, n):\n            for j in range(i):\n                if nums[j] < nums[i]:\n                    dp[i] = max(dp[i], dp[j] + 1)\n        return max(dp)",
  "verified": true,
  "teacher": "qwen2.5-32b",
  "sample_idx": 0
}
```

字段说明：
- `verified`: 必须为 true 才进训练集（见 §5 验证过滤）。
- `teacher`: 记录来源老师，便于复现。
- `sample_idx`: Best-of-N 中第几个采样 AC 的（0=贪心首发）。

## 3. 30B 蒸馏请求格式（老师一次生成，AB 同源）

**一个提示词、一次生成，产出 Rich CodeIR + 代码**。L 由 R 删字段派生，CoT-baseline 由 R 线性化派生（§6）。

### System prompt（草稿）

```
你是算法竞赛专家。给定一道编程题，先输出结构化解题中间表示(CodeIR)，再输出对应 Python 代码。

要求：
1. CodeIR 必须先于代码，且代码严格按 CodeIR.algorithm 逐步实现，二者一致。
2. key_insight 用一句话点破本题最关键的算法观察，这是最重要的字段。
3. algorithm 是有序步骤，覆盖主流程与所有 edge_cases，详到"照着能直接翻译成代码"。
4. complexity 必须满足题目对时间/空间的隐含要求。
5. code 是可直接提交的完整 class Solution，不含任何解释性文字。
```

### Response schema（遵循全局规范：顶层结构约束，字段描述放 system prompt，避免深嵌套）

```json
{
  "type": "object",
  "properties": {
    "codeir": {
      "type": "object",
      "properties": {
        "signature":       { "type": "string" },
        "pattern":         { "type": "string" },
        "key_insight":     { "type": "string" },
        "data_structures": { "type": "array", "items": { "type": "string" } },
        "algorithm":       { "type": "array", "items": { "type": "string" } },
        "complexity":      { "type": "object",
                             "properties": { "time": {"type":"string"}, "space": {"type":"string"} } },
        "edge_cases":      { "type": "array", "items": { "type": "string" } }
      },
      "required": ["pattern", "key_insight", "algorithm"]
    },
    "code": { "type": "string" }
  },
  "required": ["codeir", "code"]
}
```

### 采样策略（item 6，自适应 Best-of-N）

```
对每题：
  先贪心 1 次 (temperature=0)  → 跑验证过滤
    AC      → 收，sample_idx=0，停
    WA/Err  → 重采样，temperature≈0.8，最多再 N=8 次
                取第一个 AC，停；全 WA → 丢弃该题（无干净样本）
```

## 4. 测试用例格式（LeetCode functional 风格）

LeetCode 类题为"函数调用式"：给参数、比对返回值。验证过滤与最终评测共用此格式。

```json
{
  "problem_id": "lc-300-demo",
  "entry_point": "lengthOfLIS",
  "test_cases": [
    {"input": {"nums": [10,9,2,5,3,7,101,18]}, "expected": 4},
    {"input": {"nums": [0,1,0,3,2,3]},         "expected": 4},
    {"input": {"nums": [7,7,7,7]},             "expected": 1},
    {"input": {"nums": []},                    "expected": 0}
  ],
  "timeout_sec": 10
}
```

- `entry_point`: 被调方法名，执行器据此实例化 Solution 并调用。
- `expected`: 期望返回值；全部用例通过(且不超时)才算 AC。
- `timeout_sec`: 单题执行墙钟上限，防死循环。

## 5. 验证过滤（数据净化口径）

```
输入：(题面, codeir, code) + 该题 test_cases
执行：真机跑 code，对每条 test_case 比对返回值
判定：
  全部用例通过 且 未超时  → verified=true  → 进训练集
  任一失败 / 超时 / 报错   → verified=false → 丢弃（或触发 §3 重采样）
```

> 意义：学生只学老师"做对的子集"，擦掉噪声 → pass@1 才有机会反超老师平均水平。
> 这是 LeetCode 白送的能力（对错可执行判定，不靠人评）。

## 6. 三条对照线的训练样本格式（全部由 §2 同源三元组派生）

> 关键：实验臂、baseline 的样本**都来自同一份老师生成**，只是切分/线性化方式不同 → 杜绝提示词 confound。

### 实验臂 · LoRA_A（题 → CodeIR）

```json
{
  "instruction": "<模型A system prompt：只输出 CodeIR，不写代码>",
  "input": "<题面 statement + signature>",
  "output": "<codeir 的序列化文本，如 YAML/JSON 字符串>"
}
```

### 实验臂 · LoRA_B（CodeIR → 代码）

```json
{
  "instruction": "<模型B system prompt：严格按 CodeIR 翻译成 Python，不增删逻辑>",
  "input": "<codeir 的序列化文本>",
  "output": "<code 字符串>"
}
```

### 对照臂 baseline（题 → CoT思路 + 码，单模型不拆）

由同一份 codeir **线性化成行内 CoT**（非另起提示词），保证信息量与实验臂一致：

```json
{
  "instruction": "<baseline system prompt：先思考再写代码>",
  "input": "<题面 statement + signature>",
  "output": "<将 codeir 各字段拉平成自然语言思路段落>\n\n<code 字符串>"
}
```

> 线性化规则（codeir → CoT 文本）固定、可脚本化，确保"同源、等信息、只差结构"。
> baseline LoRA rank=16（=A+B），容量对齐。

## 7. 数据集目录约定（草稿，磁盘敏感）

```
data/
  raw_problems/        # 题面 + 测试用例（老题训练 / 新题测试 分目录）
  teacher_raw/         # 30B 原始生成（含未过滤的 WA，便于复盘；可定期清理省盘）
  verified_triples/    # verified=true 的干净三元组（训练源头）
  sft_armA/  sft_armB/ sft_baseline/   # 三条线各自 SFT 样本（由 verified_triples 派生）
  test_livecodebench/  # 时间切分新题测试集
```

> 51G 磁盘提醒：`teacher_raw` 会膨胀（每题最多 9 次采样），过滤后及时清理，只长期保留 `verified_triples`。
